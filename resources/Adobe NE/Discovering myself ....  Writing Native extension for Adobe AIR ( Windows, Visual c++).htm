Writing Native extension for Adobe AIR ( Windows, Visual c++)


Hi Friends,
In this post I am going to show you how to write a basic native extension and send messages via FREObject and dispatching events. Please see the attached folder (http://dl.dropbox.com/u/79189962/NativeCode.zip) for complete code , it contains following files :


1) One project folder for native cpp code (NativeExtensionSample).
2) One Project folder for action script library (NativeExtensionLib).
3) One Project folder for Action Script app which will use native extension (nativeApplicationApp).
Apart from these 3 folders we need following files:
1) extension.xml
2) abhisek.p12 (this will be used for signing purpose, you need to create your own )
Before jumping to the native cpp code and action script library code, let’s have a look at content of extension.xml.
< extension xmlns="http://ns.adobe.com/air/extension/3.1" >
                < id >com.adobe.NativeExtension< /id >
                < versionNumber >1.0.0< /versionNumber >
                < platforms >
                                < platform name="Windows-x86" >
                                <  applicationDeployment  >
                                                <  nativeLibrary  > NativeExtensionSample.dll < /nativeLibrary >
                                                < initializer > ExtInitializer < /initializer >
                                                < finalizer >ExtFinalizer< /finalizer >
                                < /applicationDeployment >
                                < /platform >
                < /platforms >
< /extension  >
"id" is  the extension id , we will use this id in action script project too, under  native Library  section we specify the name of the native dll and we need to declare our native side initializer and finalizer method name here.
Now let’s begin then with writing a native code:
To write a native code, create an empty dll project in Visual Studio (Visualc++ ->Win32->Win32 Project, then check DLL and empty project).  Now we can write a sample native cpp application, as we have declared in extension.xml ,the entry point will be ExtInitializer function and inside this function we will declare other methods which we are going to use.
// Initialization function of each extension
__declspec(dllexport) void ExtInitializer(void** extDataToSet, FREContextInitializer* ctxInitializerToSet, FREContextFinalizer* ctxFinalizerToSet) {
        printf("inside extension initializer method");

        *extDataToSet = NULL;
        *ctxInitializerToSet = &ContextInitializer;
        *ctxFinalizerToSet = &ContextFinalizer;
}
Few important things I want to highlight here,
1) You need to export your method by either using  __declspec(dllexport)  or using .def file. In this example I have used __declspec(dllexport) way.
2) You need to include FlashRuntimeExtensions.lib inside your project, this can be done via copying this lib to any folder and specifying the path of that folder in Linker->General->Additional Library Directories. After this specify the lib name in Linker->Input->Additional Dependencies.
3) You can send and receive data from action script by using FREObject only. In our case we are sending a string using FREObject.
4) ExtFinalizer will be called when our extension is unloaded.

Now let’s look at the Action Script side, In the action script side we will create one library project and we will include the swc from this library to our sample app. We can create extension context by using
extContext = ExtensionContext.createExtensionContext("com.adobe.NativeExtension","type"); // creatng extension context note we have to use the same extension id we mentioned in extension.xml
extContext.call("nativeHello",disp) as String; //calling the native side method
Now I am writing a simple method which will listen for event dispatched from native side.
public function eventMethod(event:StatusEvent):void {
                if((event.level == "status") && (event.code == "doneFromNative"))
                                dispatchEvent(new StatusEvent("doneFromNative"));
                }
Please see the attached Action Script library code for more info.
Now we just need to create a sample app and include the swc from library project and call methods of library. Like if we look inside nativeApplicationApp we are calling nativeHello() method of library which will call the nativeHello method of native code and will return a string from native side.
We can also receive the event and their code, level dispatched from native side. Like for example in our Library project we are getting these values and dispatching a new event.
Ok, so we are almost done , we just need to add extension tag inside our sample app application.xml file like this .
< extensions >
                < extensionID >com.adobe.NativeExtension< /extensionID >
  < /extensions >
< /application >  
And you need to specify extendedDesktop in supported profile.
< supportedProfiles >extendedDesktop< /supportedProfiles >
Now how to run this???
Yes, now we need to create an ane and then package it with sample app.
First unzip the .swc from library action script project and get the "library.swf" from it, you will need the original .swc file too while creating ane.
To create ane please use this command, note you need to modify adt path and all other files
C:\Users\abmishra\Desktop\MyNativeExtension > "C:\Program Files\Adobe\Adobe Flash Builder 4.6\sdks\4.6.0\bin\adt.bat" -package -tsa none -storetype pkcs12 -keystore abhisek.p12 -storepass **** -target ane NativeApplicationApp.ane extension.xml -swc NativeExtensionLib.swc -platform Windows-x86 library.swf NativeExtensionSample.dll

After this create a folder named as anefolder and put the .ane inside that folder, now we just need to create an exe and we are all done. We can create an .exe by using this command, again please modify the path and other files.

C:\Users\abmishra\Desktop\MyNativeExtension > "C:\Program Files\Adobe\Adobe Flash Builder 4.6\sdks\4.6.0\bin\adt.bat" -package -tsa none -storetype pkcs12 -keystore abhisek.p12 -storepass pass123 -target native NativeApplicationApp  nativeApplicationApp-app.xml nativeApplicationApp.swf -extdir anefolder
